멀티 에이전트

복잡한 작업을 여러 에이전트가 병렬로 분담하여 처리합니다. 토큰 소모량이 단일 에이전트 대비 많아질 수 있습니다. 비활성화 시 단일 에이전트 방식으로 동작합니다.

파일 자동 업데이트

LLM이 제안한 코드를 기반으로 파일을 자동으로 업데이트할지 여부를 설정합니다. 비활성화 시, 업데이트 전에 확인 메시지가 표시됩니다.

자동 파일 삭제

LLM이 요청한 파일 삭제를 자동으로 수행할지 여부를 설정합니다. 비활성화 시, 파일 삭제 전에 확인 메시지가 표시됩니다.

도구 자동 실행

모든 도구(파일 읽기, 검색 등)를 자동으로 실행할지 여부를 설정합니다. 비활성화하면 모든 도구 실행 전에 확인 메시지가 표시됩니다.

명령어 자동 실행

LLM 응답에 포함된 bash, powershell, cmd 명령어를 자동으로 실행할지 여부를 설정합니다. 비활성화하면 명령어 실행전전에 확인 메시지가 표시됩니다.

프로젝트 외부 파일 차단

프로젝트 디렉토리 외부의 파일 생성, 수정, 삭제, 읽기를 차단합니다. 워크스페이스 밖의 파일 조작을 방지하여 안전성을 높입니다.

자동 테스트 재시도

자동 테스트 검증(Smoke Test, Diagnostics, Lint Check)이 실패했을 때 자동으로 재시도하는 최대 횟수를 설정합니다.

회

자동 오류 수정

터미널에서 오류가 발생했을 때 자동으로 수정을 시도하는 최대 횟수를 설정합니다.

회

소스코드 자동완성

입력 중 AI가 코드를 자동 완성합니다 (Ghost Text). Tab 키로 완성을 수락합니다. 빠른 응답을 위해 AI 모델 탭의 라우팅 모델에서 전용 모델을 설정하는 것을 권장합니다.

오류 보고 전송

확장 프로그램에서 발생하는 오류를 서버로 전송하여 안정성 개선에 활용합니다.

설정 내보내기 / 가져오기

현재 설정을 JSON 파일로 내보내거나, 이전에 내보낸 파일에서 설정을 가져올 수 있습니다.

빌드/테스트 명령어

코드 생성 후 자동으로 실행할 검증/포맷 명령어를 직접 지정합니다. 비워두면 프로젝트 타입에 따라 자동 감지됩니다.
