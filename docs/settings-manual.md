# AgentGoCoder 설정 화면 사용자 매뉴얼

> 버전: v1.2.5 기준
> 설정 화면은 좌측 내비게이션의 **7개 탭**으로 구성됩니다.
> 각 섹션의 `📷` 표시 위치에 스크린샷을 넣으세요.

| 탭 | 내용 |
|---|---|
| 일반 | 언어·테마·자동화 동작·설정 가져오기/내보내기 |
| AI 모델 | 엔진 선택, 모델별 옵션, Ollama, 모델 라우팅, 사용자 정의 모델 |
| 빌드/테스트 | 코드 생성 후 자동 실행할 검증/포맷 명령어 |
| Hot Load | 사용자 입력 키워드에 반응하는 자동 명령 실행 |
| Skills | AI가 따를 개발 규칙·정책 Markdown 관리 |
| 보안 | 컨텍스트 제외 패턴, 도구 실행 보안 규칙 |
| 사용량 | 메모리·세션·LLM 호출 통계 |

### 2-2. 모델 설정 (선택한 클라우드 프로바이더 카드)

> 메인 드롭다운에서 클라우드 프로바이더를 선택하면 "**{프로바이더} 설정**" 카드가 나타납니다.

📷 _[모델 설정 카드 스크린샷 — 직접 입력 포함]_

| 항목 | 설명 | 비고 |
|---|---|---|
| **모델 드롭다운** | 해당 프로바이더의 프리셋 모델 목록에서 선택합니다. | 예: Claude Opus 4.8, GPT-5.4, Gemini 3.5 Flash |
| **+ 직접 입력…** | 드롭다운 맨 아래 항목. 선택하면 모델 ID 입력칸이 나타납니다. | 프리셋에 없는 새 모델용 |
| └ 모델 ID 직접 입력 | 원하는 모델 ID를 직접 입력합니다. 엔드포인트·인증은 선택한 프로바이더 설정을 그대로 사용합니다. | 예: `claude-opus-4-8` |
| └ 이 모델 적용 | 입력한 모델 ID를 활성 모델로 적용합니다. 새 LLM 버전이 나와도 확장 업데이트 없이 사용 가능합니다. | 버튼 |
| **API 키** | 해당 프로바이더의 API 키를 입력하고 **API 키 저장**을 누릅니다. 같은 프로바이더의 프리셋·직접입력 모델이 공유합니다. | 인증 방식이 "없음"이면 숨김 |

> 참고: OpenAI 추론 모델(`gpt-5`·`o3` 등)은 자동으로 `max_completion_tokens`를 사용하며 `temperature`/`top_p`는 생략됩니다. 일반 모델(`gpt-4o` 등)은 기존 방식대로 동작합니다.

📷 _[모델 라우팅 스크린샷]_

## 4. Hot Load

> 사용자 입력이 등록한 키워드/설명과 유사하면 자동으로 명령어를 실행합니다. **Hot Load는 다른 모든 지시보다 우선**합니다.

📷 _[Hot Load 탭 스크린샷]_

| 항목 | 설명 | 옵션 / 예시 |
|---|---|---|
| **+ Hot Load 추가** | 새 Hot Load 등록 폼을 엽니다. | 버튼 |
| └ 키워드 | 트리거가 될 단어들(쉼표 구분). | 예: `빌드, build, 컴파일` |
| └ 동작 설명 | 어떤 상황에서 실행할지 설명. | 예: `프로젝트를 빌드할 때 실행` |
| └ 실행 명령어 | 트리거 시 실행할 명령어. | 예: `npm run build` |
| └ 완료 조건 | 성공 판정 기준. | 없음 / 종료 코드 일치 / 출력 문자열 포함 / 출력 문자열 미포함 / 파일 존재 |
| └ 조건 값 | 완료 조건에 필요한 값(코드·문자열·경로). | 조건 선택 시 표시 |
| └ 재시도 횟수 | 실패 시 재시도할 횟수. | 0 ~ 10 / 기본 0 |
| └ 실패 시 동작 | 최종 실패 시 처리. | 중단 / LLM에 에러 전달 |
