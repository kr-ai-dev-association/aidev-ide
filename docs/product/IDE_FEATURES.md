# CodePilot IDE — 주요 기능 상세

> VS Code 기반 AI 개발 어시스턴트. 단순 채팅을 넘어 코드 생성·파일 수정·명령 실행·테스트까지 개발 워크플로우 전체를 자동화합니다.

---

## 1. 계정 및 조직 연결

### 기능 설명
- **OAuth 로그인**: Google / GitHub 계정으로 1클릭 로그인 (VS Code URI 핸들러 기반)
- **조직 가입**: 관리자에게 받은 API 키(`cpk_...`)를 입력하면 즉시 조직에 소속되어 관리자 설정이 자동 동기화됩니다
- **프로필 표시**: 이메일, 조직명, 플랜, 만료일 확인
- **설정 동기화**: 조직 설정을 수동으로 즉시 갱신하는 동기화 버튼 제공

### 특이사항
- 설정 동기화는 로그인 시 자동 수행 + 5분 주기 갱신 + 오프라인 globalState 캐시 보장
- 조직 탈퇴 시 관리자 설정 해제, 개인 설정으로 복귀

## 2. 일반 설정

| 설정 | 내용 |
|------|------|
| **언어** | 한국어 / 영어 / 중국어 / 스페인어 / 독일어 / 프랑스어 / 일본어 |
| **테마** | 다크(초록 강조) / 라이트(파란 강조) / 자동(VS Code 테마 연동) |
| **멀티 에이전트** | 복잡한 작업을 병렬 서브에이전트로 분산 처리 (기본 OFF) |
| **파일 자동 업데이트** | LLM 코드 제안을 파일에 즉시 반영 (OFF 시 확인 대화상자) |
| **파일 자동 삭제** | LLM 삭제 요청을 즉시 실행 (OFF 시 확인 대화상자) |
| **도구 자동 실행** | 파일 읽기·검색 등 모든 도구 자동 실행 (OFF 시 승인 요청) |
| **MCP 도구 자동 실행** | MCP 서버 도구 자동 실행 (OFF 시 승인 요청) |
| **명령 자동 실행** | bash/PowerShell 명령 즉시 실행 (OFF 시 확인 대화상자) |
| **자동 테스트 재시도** | 실패한 자동 테스트 최대 재시도 횟수 (1~10, 기본 3) |
| **자동 에러 수정** | 터미널 에러 자동 수정 최대 재시도 횟수 (1~10, 기본 3) |
| **에러 보고** | 확장 오류를 백엔드로 전송하여 안정성 개선에 활용 |

> **조직 소속 사용자**: 관리자가 `required`로 설정한 항목은 변경 불가, `recommended`는 개인 비활성화 가능

## 3. AI 모델

### 메인 모델 선택
- **Ollama (로컬/리모트)**: 사내 서버나 PC에 설치한 오픈소스 모델 직접 연결
- **관리자 제공 모델**: 조직 관리자가 등록한 클라우드 API 모델 (OpenAI, Claude, 등)
- **스트리밍 응답**: 실시간 토큰 스트리밍 ON/OFF

### Ollama 설정
| 항목 | 내용 |
|------|------|
| **서버 유형** | 로컬(localhost) / 리모트(원격 서버) |
| **API URL** | 기본값 `http://localhost:11434` |
| **엔드포인트** | `/api/generate` 또는 `/api/chat` |
| **모델명** | 설치된 모델 자동 감지 또는 직접 입력 (예: `gemma3:27b`) |

### 모델 라우팅 (작업별 전용 모델)
| 역할 | 설명 | 권장 모델 |
|------|------|-----------|
| **Compactor** | 긴 대화를 요약·압축 | 저비용 경량 모델 |
| **Command** | 터미널 명령어 생성 | 코드 특화 모델 |
| **Intent** | 사용자 의도 분류 | 빠른 경량 모델 |

각 역할 모델이 미설정 시 메인 모델로 자동 폴백

## 4. MCP 서버 (Model Context Protocol)

### 기능
- **관리자 MCP**: 조직 관리자가 등록한 서버 (읽기 전용 표시, 연결/해제만 가능)
- **개인 MCP**: 사용자가 직접 추가하는 커스텀 서버

### 연결 방식
| 유형 | 설명 | 예시 |
|------|------|------|
| **로컬(stdio)** | 로컬 프로세스 실행 | `npx -y @anthropic/mcp-server-weather` |
| **리모트(HTTP)** | 외부 MCP 서버 URL 연결 | `http://your-server/mcp` + API 키 |

### 커스텀 프롬프트
각 서버에 "이 도구를 언제 사용하라"는 지시를 추가해 LLM이 올바르게 활용하도록 유도

> **관리자 설정 vs 기본 설정**:
> `enforcement: required / recommended` → **관리자 설정** (조직 등록)
> `enforcement: preset` → **기본 설정** (시스템 기본 제공)

## 5. RAG (검색 증강 생성)

- **조직 RAG 소스**: 관리자가 업로드한 문서 소스 (읽기 전용 표시)
- **개인 RAG 소스**: 사용자 직접 업로드
- **지원 포맷**: PDF, TXT, MD, JSON, CSV, DOCX
- **동작**: 사용자 질문 시 해당 소스에서 유사 문서를 검색하여 LLM 컨텍스트에 주입

## 6. 빌드/테스트

### 역할
코드 수정 후 LLM이 자동으로 실행하는 검증 명령어를 정의합니다.

| 항목 | 내용 |
|------|------|
| **Validation** | 타입 체크, 린트, 컴파일 검증 |
| **Formatter** | 코드 포맷 적용 |
| **언어 타겟** | TypeScript, Python, Java, Go, Rust, C# 등 또는 전체 |
| **미설정 시** | 프로젝트 타입 자동 감지 후 기본 검증 수행 |

> 관리자 설정이 있으면 개인 설정보다 **관리자 설정이 우선** 실행

## 7. Hot Load

### 역할
특정 키워드를 입력하면 지정된 명령어를 **최우선으로 자동 실행**합니다.
일반 LLM 지시보다 높은 우선순위를 가집니다.

| 항목 | 내용 |
|------|------|
| **키워드** | 쉼표 구분 트리거 (예: `build, compile, 빌드`) |
| **명령어** | 실행할 셸 명령 |
| **완료 조건** | 없음 / exit code / 출력 매칭 / 파일 존재 |
| **재시도** | 실패 시 최대 재시도 횟수 (0~10) |
| **실패 동작** | 중지 / LLM에 에러 전달 |

## 8. Skills (개발 규칙)

### 역할
마크다운(.md) 파일을 업로드해 LLM이 코드를 생성할 때 **항상 참조하는 규칙**을 정의합니다.
파일은 `./.agent/rules/` 디렉토리에 자동 저장됩니다.

| 카테고리 | 내용 |
|----------|------|
| **stable-version** | 프로젝트 버전 관리 정책 |
| **coding-style** | 코드 컨벤션, 네이밍, 아키텍처 패턴 |
| **project-architecture** | 컴포넌트·모듈 구조, 디자인 패턴 |
| **dependency-policy** | 라이브러리 관리, 버전 제약, 보안 요구사항 |
| **db-policy** | 스키마 가이드라인, 마이그레이션, 쿼리 최적화 |

## 9. 보안

### A. 컨텍스트 제외 패턴
LLM 검색·인덱싱에서 제외할 파일/폴더 지정 (glob 문법)
- **기본 제외**: `.env`, `.git`, `node_modules`, `dist` 등 62개 내장
- **커스텀 추가**: 개인 또는 조직 단위로 패턴 추가 가능

### B. 도구 실행 보안 규칙

| 규칙 유형 | 설명 | 예시 |
|-----------|------|------|
| **명령 차단** | 위험한 명령 실행 방지 | `rm -rf /`, `docker rm`, `kubectl delete` |
| **파일 보호** | 지정 파일 수정/삭제 차단 | `.env`, `.aws/credentials`, `package.json` |
| **숨김 파일** | 민감 파일이 컨텍스트에 노출되지 않도록 차단 | `.ssh/`, `.aws/` |

- 관리자가 `required`로 설정한 규칙은 사용자 비활성화 불가
- 정규식 패턴 지원으로 유연한 매칭

## 11. 채팅 인터페이스

### 턴 시스템 & Undo
- 각 사용자-AI 교환 = 1턴으로 저장
- **체크포인트 스택**: 턴별 파일 변경 스냅샷 누적
- **Undo 버튼**: 최신 턴부터 순차적으로 되돌리기 (Cascade Undo)

### 코드 블록 기능
- 구문 강조 (highlight.js)
- **Copy 버튼**: 코드 복사
- **Run 버튼**: 터미널에서 즉시 실행 → 실행 중 **Stop 버튼**으로 전환

### Thinking Bubble
- LLM의 내부 추론 과정 실시간 표시
- 2줄 요약 + 펼치기/접기 토글

### 인라인 Diff 뷰어
- 파일 변경 사항을 라인별 강조 표시
- **Keep** (수락) / **Discard** (거부) 선택

### 토큰 게이지
- 현재 컨텍스트 토큰 사용량 실시간 표시
- 한계 접근 시 경고/위험 색상 변경

## 12. 멀티 에이전트 오케스트레이션

복잡한 작업을 여러 에이전트가 병렬로 분담 처리하는 시스템 (설정 ON 시 활성화)

```
사용자 요청
  ↓
TaskSplitter (LLM) — 독립 서브태스크로 분해
  ↓
SubAgentLoop × N — 병렬 실행 (최대 3개 동시)
  ↓
ResultMerger — 결과 통합 → 최종 응답
```

- 단순 작업(≤2 서브태스크) 또는 에러 시 → 단일 에이전트로 자동 폴백
- 읽기 전용 도구는 `Promise.all`로 병렬 실행 (쓰기 도구는 순차 실행 유지)

## 13. 자동 Import 정리 (파일 삭제 연동)

LLM이 파일을 삭제하면:
1. ripgrep으로 해당 파일을 `import`하는 소스 파일 자동 검색
2. 검출된 파일 목록을 LLM 컨텍스트에 주입
3. LLM이 자동으로 import 구문을 정리 (dead import 제거)
