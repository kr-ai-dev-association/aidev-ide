# CodePilot Admin — 주요 기능 상세

> 조직 관리자가 전체 팀의 AI 사용 정책을 설정하는 웹 대시보드.
> 여기서 설정한 정책은 모든 팀원의 IDE에 자동으로 적용됩니다.

---

## 1. AI 모델 관리 (`/ai-model`)

### 기능
팀에서 사용할 AI 모델을 등록하고, 사용 여부와 강제 적용 수준을 결정합니다.

### 모델 등록 항목
| 필드 | 내용 |
|------|------|
| **이름** | 팀 내 식별용 이름 |
| **Provider** | `chat_completions` / `text_completion` |
| **Model ID** | 실제 모델 식별자 (예: `gpt-4o`, `claude-3-5-sonnet`) |
| **Endpoint URL** | 커스텀 API URL (OpenAI compatible) |
| **API Key** | 조직 공용 API 키 |
| **Context Window** | 최대 입력 토큰 수 |
| **Max Output Tokens** | 최대 출력 토큰 수 |
| **Temperature / TopP** | 응답 다양성 조절 |
| **Auth Type** | Bearer / API-Key 헤더 방식 |
| **Custom Headers** | JSON 형태의 추가 헤더 |
| **Streaming** | 스트리밍 응답 지원 여부 |

### 적용 수준
- `required`: IDE에서 해당 모델만 사용 강제 (사용자가 변경 불가)
- `recommended`: IDE에 기본 선택 제안 (사용자가 다른 모델로 변경 가능)

### 내장 프리셋
관리자가 추가 구성 없이 바로 활성화할 수 있는 검증된 모델 템플릿 제공

## 2. MCP 서버 관리 (`/mcp-server`)

### 기능
사내 전용 도구나 외부 서비스 연동을 위한 MCP(Model Context Protocol) 서버를 관리합니다.

### 서버 등록 항목
| 필드 | 내용 |
|------|------|
| **이름** | 서버 식별 이름 |
| **연결 방식** | `stdio` (로컬 프로세스) / `http` (원격 URL) |
| **stdio - 명령** | 실행 명령 (예: `npx`) |
| **stdio - 인수** | 명령 인수 (예: `-y, @anthropic/mcp-server-weather`) |
| **http - URL** | 원격 MCP 서버 URL |
| **http - API Key** | 원격 서버 인증 키 (선택) |
| **커스텀 프롬프트** | LLM이 이 서버 도구를 사용하는 방법에 대한 지시 |
| **활성화** | 서버 ON/OFF |

### 적용 수준
- `required`: 사용자가 비활성화 불가
- `recommended`: 사용자가 선택적 비활성화 가능

### IDE에서 표시 방식
- `required` / `recommended` → **관리자 설정** 섹션에 표시
- `preset` (시스템 기본 제공) → **기본 설정** 섹션에 표시

## 3. 빌드/테스트 명령 (`/build-test-config`)

### 기능
코드 수정 후 LLM이 자동으로 실행하는 검증 명령어를 조직 단위로 정의합니다.

### 명령 등록 항목
| 필드 | 내용 |
|------|------|
| **유형** | `validation` (검증) / `formatter` (포맷) |
| **언어** | TypeScript, Python, Java, Go, Rust, C# 등 또는 전체 |
| **설명** | 명령의 목적 설명 |
| **명령어** | 실제 실행 셸 명령 (예: `tsc --noEmit`, `npm run lint`) |

### 특이사항
- 관리자 설정이 있으면 개인 설정보다 **관리자 설정 우선 실행**
- 설정 없으면 프로젝트 유형 자동 감지 후 기본 명령 실행

---

## 4. Hot Load 설정 (`/hotload`)

### 기능
특정 키워드가 포함된 사용자 입력을 감지하면 지정 명령을 **최우선으로 자동 실행**합니다.

### 등록 항목
| 필드 | 내용 |
|------|------|
| **키워드** | 트리거 단어들 (쉼표 구분) |
| **설명** | 이 Hot Load의 목적 |
| **명령어** | 실행할 셸 명령 |
| **완료 조건** | 없음 / exit code / 출력 매칭 / 파일 존재 |
| **재시도 횟수** | 실패 시 재시도 (0~10) |
| **실패 동작** | 중지 / LLM에 에러 전달 |

### 우선순위
Hot Load는 모든 LLM 지시보다 높은 우선순위를 가짐

## 5. Skills (개발 규칙) (`/skills`)

### 기능
마크다운 파일을 업로드하면 IDE에서 LLM이 코드를 생성할 때 항상 이 규칙을 참조합니다.

### 카테고리
| 카테고리 | 내용 |
|----------|------|
| `stable-version` | 프로젝트 버전 관리 정책 |
| `coding-style` | 코드 컨벤션, 네이밍 규칙, 아키텍처 패턴 |
| `project-architecture` | 컴포넌트·모듈 구조, 디자인 패턴 |
| `dependency-policy` | 라이브러리 관리, 버전 제약, 보안 요구사항 |
| `db-policy` | 스키마 가이드라인, 마이그레이션, 쿼리 최적화 |

### 파일 관리
- 카테고리당 다수의 `.md` 파일 업로드 가능
- 적용 수준(required/recommended)별 구분 가능

## 6. 보안 규칙 (`/security-rules`)

### A. 명령 차단 (blocked_command)
위험한 LLM 생성 명령의 실행을 원천 차단합니다.

**기본 내장 규칙 (62개 중 일부)**:
- `rm -rf /` (전체 삭제)
- `docker rm`, `docker rmi` (컨테이너/이미지 삭제)
- `kubectl delete` (K8s 리소스 삭제)
- `sudo rm`, `sudo dd` (관리자 삭제 명령)
- `git push --force` (강제 푸시)
- `DROP TABLE`, `DROP DATABASE` (DB 삭제)

### B. 파일 보호 (protected_file)
LLM이 특정 파일을 수정하거나 삭제하지 못하도록 차단합니다.

**기본 내장 규칙 (일부)**:
- `.env`, `.env.*` (환경 변수)
- `.aws/credentials`, `.ssh/` (인증 정보)
- `package.json`, `package-lock.json` (의존성 명세)

### C. 숨김 파일 (hidden_file)
민감 파일이 LLM 컨텍스트에 포함되지 않도록 인덱싱에서 제외합니다.

### 공통
- 정규식 패턴 지원으로 유연한 매칭
- `required`: 사용자 비활성화 불가
- `recommended`: 사용자 선택적 비활성화 가능
- 내장 프리셋 개별 비활성화 가능

## 7. 제외 패턴 (`/exclude-patterns`)

### 기능
코드 인덱싱 및 LLM 검색에서 제외할 파일/폴더 패턴을 관리합니다.

### 기본 제외 패턴 (일부)
- `node_modules/`, `.git/`, `dist/`, `build/`
- `.env`, `*.log`, `*.lock`
- `__pycache__/`, `.venv/`, `.idea/`, `.vscode/`

### 커스텀 패턴
- glob 문법 지원 (예: `secrets/`, `my-private-data/**`)
- 조직 단위로 추가 → 팀 전체에 적용

## 8. RAG 관리 (`/rag`)

### 기능
팀 공용 문서 소스를 구축하여 LLM이 질문에 답할 때 해당 문서를 참조하게 합니다.

### 소스 관리
| 기능 | 내용 |
|------|------|
| **소스 생성** | 이름/설명과 함께 새 RAG 소스 생성 |
| **문서 업로드** | PDF, TXT, MD, JSON, CSV, DOCX 지원 |
| **재인덱싱** | 벡터 임베딩 재구축 |
| **소스 복제** | 기존 소스 복사 |

### 표시 항목
- 소스명, 설명, 문서 수, 벡터 수

## 12. 접근 로그 & 에러 로그 (`/logs`)

### 접근 로그
- 로그인/로그아웃, API 호출, 설정 변경, 도구 실행 이벤트 기록
- 날짜·사용자·액션별 필터링
- CSV 내보내기

### 에러 로그
- LLM API 오류, MCP 연결 실패, 도구 실행 에러, 파일 작업 오류 수집
- 타임스탬프·사용자·컴포넌트·메시지 상세 표시
- 필터링 및 CSV 내보내기
